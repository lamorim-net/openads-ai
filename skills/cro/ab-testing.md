# A/B Testing Skill

## When to Use This Skill

Activate when the user is planning an experiment, analyzing A/B test results, or trying to improve conversion rates.

---

## Experiment Design
Never run an A/B test without:
1. **A clear hypothesis**: "If we [change X], then [metric Y] will improve because [rationale]."
2. **A single variable**: Don't change the headline, hero image, and button color all at once. You won't know what caused the lift.
3. **Sufficient traffic**: Testing with 50 visitors is meaningless. Use a sample size calculator.

## Statistical Significance
- Don't stop a test early just because it looks like it's winning.
- Wait for at least 95% statistical significance.
- A test should run for at least 1-2 full business cycles (usually 7-14 days) to account for day-of-week anomalies.

## What to Test First
If traffic is low, test macro changes (the whole offer, the pricing model, the core value prop). Micro changes (button color) require massive traffic to reach significance.
